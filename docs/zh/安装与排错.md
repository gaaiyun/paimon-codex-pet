# 安装与排错

## 安装步骤

在仓库根目录运行：

```powershell
Copy-Item -Recurse -Force .\pet\paimon "$env:USERPROFILE\.codex\pets\paimon"
```

然后重启 Codex，或在命令菜单里执行 `Force Reload Skills` / reload assets。

## 文件结构

```text
%USERPROFILE%\.codex\pets\paimon\
  pet.json
  spritesheet.png
```

`pet.json` 必须指向：

```json
{
  "spritesheetPath": "spritesheet.png"
}
```

## 已知兼容性

最初的版本使用 `spritesheet.webp`。部分 Codex 桌面版本可以在设置里发现 pet，但选择后无法成功加载。当前版本改为 PNG 作为正式加载文件，并保留 WebP 作为归档素材。

## 验证命令

仓库自带一个校验脚本，只依赖 Python 标准库（装了 Pillow 会做更严格的检查，但不是必需）。在仓库根目录运行：

```powershell
python tools\validate_atlas.py
```

它会按 `assets/manifest.json` 里记录的规格，检查正式 spritesheet 是否为 1536x1872、RGBA、8x9 atlas、192x208 单格，并确认每个状态帧和表情帧都存在。预期输出最后一行是：

```text
Result: ok
```

也可以校验任意一张 spritesheet：

```powershell
python tools\validate_atlas.py 路径\spritesheet.png
```
