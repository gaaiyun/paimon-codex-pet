# 派蒙科研打工人 Pet v2 设计

## 目标

把现有像素风 Paimon Pet 全量重做成适合黑色 Codex 主题的高级卡通贴纸风。角色应当可爱、搞怪、精细，并像一个同时写论文与写代码的小型搭档。9 个官方状态保持同一脸型、比例、服装、眼镜和道具；动作变化更丰富，但不改变 Codex 应用控制的播放速度。

## 官方合约

- 透明 PNG 或 WebP，精确为 1536×1872，文件不超过 20 MiB。
- 8 列×9 行 atlas，单格 192×208。
- 行与帧数固定为：`idle` 6、`running-right` 8、`running-left` 8、`waving` 4、`jumping` 5、`failed` 8、`waiting` 6、`running` 6、`review` 6。
- 桌面端状态语义以 Running、Needs input、Ready、Blocked 为准；Reduced Motion 开启时使用静态帧。
- `pet.json` 不声明帧率。播放速度由应用控制，本项目不注入加速逻辑。

来源：<https://learn.chatgpt.com/docs/pets?surface=app>

## 方案比较

1. **整套重建（采用）**：重新生成 canonical base 与全部状态行。身份一致性控制工作最多，但能彻底去掉旧版像素边缘、紫色描边和风格漂移。
2. **旧 atlas 局部修补**：只替换表情和少数动作。速度最快，但新旧画风会在状态切换时跳变，不符合“一致性强”的要求。
3. **混合重绘**：保留旧轮廓并进行高分辨率卡通化。可复用旧动作节奏，但轮廓与配件细节仍受旧像素设计限制。

## 角色视觉锁

- 形象：非像素的 chibi cartoon / sticker，轮廓干净，适合 192×208 小尺寸阅读。
- 固定特征：白色短发、星形发饰、金色光环、半框眼镜、发间铅笔、白金黑服装、极少量冰青点缀。
- 固定道具：紧凑的黑金小键盘；带盖黑色咖啡杯固定在光环结构中，移动时不会无故漂浮或洒出。
- 色彩：珍珠白、暖象牙、香槟金、石墨黑、少量冰青。禁止紫色、粉色和霓虹蓝。
- 画面：无文字、UI、场景、地面阴影、速度线、松散粒子、辉光和无法干净抠图的软特效。
- 参考图：`G:\CodexDreamSkin\assets\paimon-research-half-rim-2560x1440.png` 只用于角色、材质和情绪参考，不照搬背景与动态残影。

## 状态动作语法

- `idle`：呼吸、眨眼、眼镜微调、发梢与披风轻摆；安静但不僵死。
- `running-right` / `running-left`：方向明确的轻快小跑，步态交替，键盘与角色作为一个整体移动；不画速度线和尘土。左右各自生成，避免眼镜、铅笔与配件镜像错误。
- `waving`：先注意到用户，再挥手、眨眼、得意收手；只用肢体表现挥手。
- `jumping`：蓄力、腾空、开心峰值、落下、站稳；只用身体高度变化表现。
- `failed`：眼镜滑低、肩膀垮下、抱住键盘或小幅委屈；不使用红叉和漂浮标点。
- `waiting`：双手托腮、看向用户、轻敲键盘边缘、礼貌催促；对应 Needs input，而不是普通待机。
- `running`：高效打字、左右扫视、短暂停顿思考、继续敲键盘；这是“任务正在执行”，不是原地赛跑。
- `review`：扶眼镜、前倾核对、轻微皱眉、确认后小点头；表现论文与代码审阅，不新增纸张、放大镜或 UI。

## 生成与数据流

1. 在 G 盘创建独立 run，保留参考图、prompt、生成条带、单帧、QA 与最终包。
2. 先生成 canonical base，确认角色在 Pet 尺寸下仍能读出眼镜、铅笔、咖啡杯与键盘。
3. 先验证 `idle` 和 `running-right`，再按状态逐行生成。每行同时引用 canonical base 和该行布局 guide。
4. 使用 `hatch-pet` 的确定性脚本提取帧、检查组件、合成 atlas、生成 WebP、contact sheet 与 GIF。
5. 只修复失败的最小状态行，不用局部补丁掩盖身份漂移。

## 安装、兼容与回滚

- 最终本机包为 `G:\codex-home\pets\paimon\pet.json` 与 `spritesheet.webp`。
- 替换前将旧包完整复制到 G 盘带时间戳的备份目录，比较文件数量与 SHA-256 后再覆盖。
- 不修改 Codex 程序文件、配置、帧率或系统 Reduced Motion 设置。
- 不自动重启 Codex；安装完成后由用户自行刷新或重启。
